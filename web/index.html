The approach

Graph-Based Genetic Optimization for Vehicle Routing via Giant Tour Decomposition. Classical GA crossovers copy contiguous segments syntactically and enforce feasibility with costly repair. This solver instead operates on giant tours (a permutation of all customers with no route delimiters) and defers feasibility to a graph-theoretic splitting step.

Giant tour representation

The GA evolves a population of giant tours with selection and mutation. Fitness is never evaluated on the tour directly — each individual is decoded into a feasible CVRP solution via the split.

Graph-based split into feasible routes

For each tour a directed acyclic graph models every feasible split: an arc i→j is a route covering customers i+1…j within vehicle capacity, weighted by its exact routing cost (depot connections included). Optimal splitting reduces to a shortest path on this graph.

Crossover & embedded local search

Classical cut-point crossover is replaced by a shortest-path-driven inheritance mechanism: the graph explicitly decides which components each parent contributes. Intra- and inter-route local search (2-opt, swap, left/right shift) runs inside the graph rather than as a separate phase. Built in Java 23 with multi-threaded graph construction and evaluation.
